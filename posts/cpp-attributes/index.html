    Attributes in C/C++

    December 12, 2021

    __attribute__ #

    In C/C++, one can apply an attribute to a function or a variable using a keyword __attribute__.

    While it is not part of ISO C standard, popular compilers such as Clang and GCC support it. So, an attribute supported by these two compilers need not be modified.

    [[attribute]] #

    There is another syntax for attributes that is introduced since C23 and C++11, which is [[attribute]].

    Attribute Namespace #

    Unlike __attribute__, this syntax requires a namespace to be specified for non-standard attributes as below:

    [[gnu::nodiscard]] int func1() { return 0; }

    GCC stores its attributes under the namespace called gnu. This is shown in its source code:

    /* Put all the GNU attributes into the "gnu" namespace.  */
register_scoped_attributes (attribute_tables[i], "gnu");

    It seems like Clang uses both its own namespace and GCC’s.

    As shown in this live demo, Clang accepts [[gnu::noinline]] while it rejects [[clang::noinline]].

    Note that there is a new syntax for an attribute namespace since C++17.

    Caveat #

    When [[attribute]] is applied to a function, it can be specified either before the return type or after the function name as below:

    [[nodiscard]] int func1() { return 0; }
int [[nodiscard]] func2() { return 0; }   // invalid
int func3 [[nodiscard]] () { return 0; }
int func4 () [[nodiscard]] { return 0; }  // invalid

    Live demo

    As shown in the above live demo, GCC 11.2 accepts the code though it emits errors while Clang 13 rejects it.

    An attribute to a lambda was introduced in C++23, and it can only be specified in one location:

    int i = [] [[nodiscard]] {
    return 0;
}();

    Live demo

    GCC with -std=c++20 accepts the code without a warning while Clang with -std=c++20 accepts the code with a warning.
